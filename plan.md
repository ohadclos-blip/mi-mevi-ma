# Plan: מי מביא מה (BringIt)
## תוכנית פיתוח — 4 שלבים

---

## שלב 1: תשתית + זרימת המארגן הבסיסית

**מה בונים:** הפרויקט מאפס עד לנקודה שבה מארגן יכול להתחבר, ליצור אירוע, ולקבל קישור.

### משימות
- [ ] אתחול פרויקט Next.js 15 + Tailwind CSS + Firebase
- [ ] הגדרת Firebase project: Firestore, Auth (Google Provider)
- [ ] דף התחברות עם Google (`/`) — כפתור "התחבר עם Google"
- [ ] יצירת document ב-`users/` בהתחברות ראשונה
- [ ] דף Dashboard בסיסי (`/dashboard`) — מוגן ב-auth guard, מציג שם המשתמש
- [ ] טופס יצירת אירוע (`/dashboard/events/new`):
  - שם, תאריך, תיאור
  - בחירת מצב (prescribed / open)
  - הגדרות: allowUnclaim, visibilityMode
  - שדות prescribed: הוספת פריטים + כמויות
- [ ] כתיבת האירוע ל-Firestore (`events/` + `items/`)
- [ ] הפניה לאחר יצירה לדף ניהול האירוע (`/dashboard/events/:id`)
- [ ] דף ניהול אירוע — מציג את הקישור הייחודי לשיתוף
- [ ] Firestore Security Rules בסיסיות (יצירה מחייבת auth)

### בדיקה ידנית ✅
> **"יצרתי אירוע וקיבלתי קישור"**
>
> 1. פתח את האפליקציה בדפדפן
> 2. לחץ "התחבר עם Google" — ודא שאתה מגיע ל-Dashboard עם שמך
> 3. לחץ "אירוע חדש", מלא פרטים, הוסף 2-3 פריטים, לחץ "צור"
> 4. ודא שאתה מועבר לדף ניהול האירוע ורואה קישור בפורמט `/event/UUID`
> 5. פתח את הקישור בטאב חדש — ודא שדף האירוע נטען (גם אם ריק בינתיים)

---

## שלב 2: זרימת האורח + בחירה בזמן אמת

**מה בונים:** האורח פותח קישור, מכניס שם, בוחר פריט — והמארגן רואה את זה מיידית.

### משימות
- [ ] דף אירוע לאורח (`/event/:id`) — Server Component לטעינה ראשונית מהירה
- [ ] NicknamePrompt — bottom sheet "מה שמך?" + שמירה ב-LocalStorage
- [ ] טעינה אוטומטית של כינוי מ-LocalStorage בביקור חוזר
- [ ] `lib/localStorage.ts` — abstraction לכל גישת LocalStorage
- [ ] הצגת רשימת פריטים:
  - prescribed: ItemRow עם כפתור "בחר" / תג "תפוס"
  - open: OpenItemEntry לתוספת פריט + הצגת פריטים קיימים
- [ ] `claimItem()` — Firestore Transaction (נעילה אופטימיסטית)
- [ ] Real-time listener (`onSnapshot`) — רשימה מתעדכנת ללא רענון
- [ ] טיפול בקונפליקט — הודעה "הפריט נתפס, בחר אחר" אם הטרנזקציה נכשלת
- [ ] הצגה לפי visibilityMode: open → שם + פריט, private → "תפוס" בלבד
- [ ] כפתור ביטול בחירה (אם allowUnclaim=true) + מחיקת claim
- [ ] Security Rules לפריטים ו-claims

### בדיקה ידנית ✅
> **"שני אורחים בוחרים פריטים בו-זמנית"**
>
> 1. פתח את קישור האירוע בשני דפדפנים שונים (למשל Chrome + Firefox)
> 2. בכל דפדפן — הכנס כינוי שונה
> 3. בדפדפן הראשון — בחר פריט ורענן את הדפדפן השני; ודא שהפריט מסומן "תפוס"
> 4. נסה לבחור את אותו הפריט מהדפדפן השני — ודא שמוצגת הודעת "הפריט נתפס"
> 5. פתח את הדשבורד בטאב נוסף — ודא שהבחירה מופיעה שם בזמן אמת ללא רענון

## שלב 3: מחזור חיי אירוע + פיצ'רי מארגן מתקדמים

**מה בונים:** הקפאה, חסימה, שכפול, התראות סיום, שיתוף לוואטסאפ.

### משימות
- [ ] דף ניהול אירוע מלא (`/dashboard/events/:id`):
  - סטטיסטיקות: כמה פריטים נלקחו מתוך כמה, progress bar
  - רשימת פריטים עם שמות הבוחרים (מארגן רואה הכל תמיד)
  - כפתורי פעולה: הקפא / חסום / מחק
  - כפתור ביטול בחירה של כל אורח (ע"י מארגן)
- [ ] לוגיקת שינוי status + עדכון Firestore
- [ ] מסך אירוע מוקפא לאורח — רשימה בקריאה בלבד + "האירוע הסתיים"
- [ ] מסך אירוע חסום לאורח — הודעה + כפתור "שלח שאלה למארגן"
- [ ] API Route `POST /api/contact-organizer` + Resend (שליחת מייל לאורח)
- [ ] טופס פניה למארגן — שם + הודעה עד 300 תווים
- [ ] כפתור "שתף לוואטסאפ" — deep link עם טקסט מוכן
- [ ] כפתור "שכפל לאירוע חדש" — מעתיק פריטים + כמויות, מאפס בחירות
- [ ] Firebase Cloud Function — `dailyEventLifecycle`:
  - סורק אירועים שתאריכם עבר
  - כותב notification ל-`notifications/`
  - הקפאה אוטומטית אחרי 7 ימים
- [ ] הצגת notifications בדשבורד המארגן — בנר עם כפתורי פעולה
- [ ] My Board (`/dashboard`) — "אירועים שאני מנהל" + "אירועים שאני משתתף"

### בדיקה ידנית ✅
> **"האירוע נחסם, האורח שולח פניה, המארגן מקבל מייל"**
>
> 1. מדף ניהול האירוע — לחץ "חסום"
> 2. פתח את קישור האירוע בדפדפן פרטי — ודא שאתה רואה מסך "האירוע נחסם"
> 3. לחץ "שלח שאלה למארגן", מלא שם והודעה, לחץ "שלח"
> 4. ודא שהמארגן קיבל מייל עם נושא `[BringIt] פניה בנוגע לאירוע: ...`
> 5. חזור לדשבורד המארגן — לחץ "שכפל", ודא שנוצר אירוע חדש עם אותם פריטים ואפס בחירות
> 6. בדוק שכפתור "שתף לוואטסאפ" פותח את וואטסאפ עם טקסט מוכן וקישור

## שלב 4: Admin Dashboard + חיבור חשבון + ליטוש

**מה בונים:** פאנל אדמין, מיזוג LocalStorage, ולוקח את האפליקציה לרמת גימור.

### משימות
- [ ] Admin guard — `ADMIN_UID` ב-env + custom claim ב-Firebase Auth
- [ ] דשבורד אדמין (`/admin`):
  - טאב אירועים: טבלה עם פילטור לפי status / תאריך
  - טאב משתמשים: טבלה עם soft-delete
  - טאב פריטים: טופ-10 פריטים נפוצים
  - טאב Audit Log: רשימת פעולות אדמין
- [ ] API Route `POST /api/admin/actions` + כתיבה ל-`audit_log`
- [ ] זרימת שדרוג אנונימי לחשבון:
  - לאחר בחירת פריט — הצעה "רוצה לשמור היסטוריה?" (bottom sheet)
  - לחיצה → Google OAuth → מיזוג LocalStorage (`migrateLocalStorageToCloud`)
  - הפניה ל-`/dashboard`
- [ ] `lib/localStorage.ts` — `migrateLocalStorageToCloud()` מלא
- [ ] דף 404 נאה לאירועים שנמחקו
- [ ] טיפול בשגיאות ב-UI — הודעות שגיאה ידידותיות
- [ ] בדיקת RTL ותצוגה על מובייל אמיתי (iPhone + Android)
- [ ] Firestore Security Rules — גרסה מלאה ומאובטחת
- [ ] הגדרת סביבות: `.env.local` (dev) + Vercel env vars (production)
- [ ] פריסה ל-Vercel + Firebase Hosting לـ Cloud Functions

### בדיקה ידנית ✅
> **"זרימה מקצה לקצה — מאורח אנונימי לחשבון, ואדמין שרואה הכל"**
>
> 1. פתח את האפליקציה בדפדפן פרטי (אנונימי)
> 2. פתח קישור אירוע, הכנס כינוי "בודק", בחר פריט
> 3. לחץ "שמור היסטוריה" → התחבר עם Google → ודא שמגיע ל-Dashboard
> 4. ב-Dashboard — ודא שרואה את האירוע תחת "אירועים שאני משתתף" עם הפריט שבחרת
> 5. עבור ל-`/admin` עם חשבון האדמין — ודא שרואה את האירוע והמשתמש בטבלאות
> 6. מה-admin — שנה status של האירוע ל-frozen — ודא שנרשמת רשומה ב-Audit Log
> 7. פתח את האפליקציה על אייפון אמיתי — ודא שהכל קריא ועובד ב-RTL
